This process is most critical when dealing with fixed licenses (named-user assignments, like a specific email address) versus floating licenses (concurrent access where a license is pulled from a pool only when in use). For a fixed license, the revocation is a specific administrative action. With a floating license, the “revocation” is often simply waiting for the user to log out or a configurable timeout period to expire. Never confuse the administrative act of revocation with a user deactivation; one is the business-level seat recovery, the other is a local software cleanup.
